# 鸿蒙不兼容的第三方库

以下为 `mobile_portainer_flutter_module` 中不支持鸿蒙（HarmonyOS）原生实现的第三方库，共 10 个。

## 需要原生平台实现的库（全部无鸿蒙支持）

| 库名 | 版本 | 缺失原因 |
|---|---|---|
| `shared_preferences` | ^2.3.0 | 需要原生 Key-Value 存储 API |
| `flutter_local_notifications` | ^20.1.0 | 需要原生通知推送 API |
| `fluttertoast` | ^9.0.0 | 需要原生 Toast/提示 API |
| `package_info_plus` | ^8.0.0 | 需要原生包信息获取 API |
| `url_launcher` | ^6.2.0 | 需要原生 URL 启动/浏览器 API |
| `mobile_scanner` | ^5.0.0 | 需要相机扫码 API |
| `path_provider` | ^2.1.0 | 需要原生文件路径 API |
| `share_plus` | ^12.0.2 | 需要原生分享面板 API |
| `permission_handler` | ^11.3.0 | 需要原生权限管理 API |
| `device_info_plus` | ^10.1.0 | 需要原生设备信息 API |

## 说明

- 上述 10 个不兼容库在项目中的包路径下均无 `ohos` / `harmonyos` / `.ohos` 平台目录
- 编译时 `flutter build har --debug` 可正常通过（Dart 层编译不依赖原生代码），但运行时调用这些插件的原生 API 会因找不到 platform channel 实现而失败
- `GeneratedPluginRegistrant.registerWith()` 的 try 块为空，证明构建工具扫描到零个有鸿蒙原生实现的插件

## 处理方式

当前已在 `main.dart` 中通过 `Platform.isAndroid` / `Platform.isIOS` 判断，在鸿蒙上跳过所有插件初始化，直接调用 `runApp()`。后续如需使用特定功能，可考虑：

1. 寻找有鸿蒙实现的替代库
2. 自行实现 `ohos` 平台通道
3. 使用纯 Dart 实现的平替方案
